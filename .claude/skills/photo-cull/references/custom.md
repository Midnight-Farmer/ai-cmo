# Profile: Custom (anything the named profiles don't fit)

Use this when the shoot doesn't map cleanly to project-reveal, wedding, event, or real-estate — a portrait/headshot session, a product shoot, a food shoot, a fashion lookbook, a fine-art set, a documentary take, or something genuinely one-off. The shared loop (render → contact sheets → curate → stage) is unchanged; what you're missing is the *curation logic*, and the user has it. Your job is to elicit it, then apply it.

## Elicit the four things that define a cull

Before culling, get these from the user (ask concisely; infer what you safely can and confirm):

1. **Goal / deliverable.** What is this set *for*? (A website portfolio? A client proof gallery? One hero image? A social set?) The purpose sets the bar.
2. **Order.** Does sequence matter (an ordered narrative/carousel) or is it an unordered set (a gallery/proof sheet)? If ordered, by what logic?
3. **Target count.** How many keepers? Total, or grouped (per subject, per outfit, per setup)?
4. **Must-haves and hard-nos.** What must be covered? What do they always cut? (Eyes-closed, hands awkward, specific angles, certain expressions, particular products.)

If the user has a **taste guide** (a file or a creed describing their eye), this is exactly where it shines — read it and let it drive the cull. See Step 0 of `SKILL.md`.

## Then apply the universal discipline

With those four answered, run the standard loop:

- **You must have looked** — every pick traces to a frame on a sheet.
- **Dedup** to the aggressiveness the goal implies (a proof gallery tolerates more near-dupes than a one-hero pick).
- **Honor the target count.**
- **If ordered, no adjacent near-duplicates; if unordered, group sensibly.**
- **Name what you benched and why.**
- **Flag** whatever the must-haves/hard-nos imply (blinks for portraits, label legibility for products, etc.).

## Common one-offs and a sensible default each

These aren't full profiles — just a starting instinct. Confirm with the user.

- **Portrait / headshot:** best N expressions per subject; flag eyes-closed, forced smiles, distracting hands. Usually unordered, grouped by subject.
- **Product:** one clean hero per SKU/angle; flag label legibility, dust, reflections. Coverage-first, like real-estate.
- **Food:** the appetizing frame per dish; flag melt/wilt/steam-gone and messy plating. Often one hero per dish.
- **Fashion / lookbook:** best frame per look/outfit; flag wardrobe malfunctions, awkward mid-stride. Grouped by look.

## Offer to make it a profile

If the user culls this same kind of shoot repeatedly, offer to capture what you learned as a new `references/<type>.md` so next time it's a named profile, not a re-elicitation. That's how the skill grows.
